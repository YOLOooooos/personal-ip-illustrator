Step 2 / Visual Candidate Sheet

为 AI 工程文章生成 6 个 IP 候选

输入文章是《为什么你的 AI 工作流总是跑不起来》。这一屏来自真实运行输出，用来让用户先看候选方向，再选择视觉 IP。

selected: Circuit Guide

Circuit Guide

robot
A
compact circuit-node robot
neutral pose + workflow board pose

在复杂 AI 工作流里帮读者找到信号路径的电路向导。

适合解释系统、失败点、可观察性和流程结构，不会变成普通真人讲师。

Log Lens

object
B
inspection lens
hovering over workflow trace

让隐藏的工作流中间状态变得可见的观察镜。

适合可观察性和调试主题，但作为长期 IP 时行动能力偏弱。

Input Gate

symbol
C
small gate symbol
sorting accepted / rejected inputs

把有效输入和不稳定噪声隔开的输入边界符号。

边界感很强，但容易只服务一个主题，不够覆盖完整工作流。

State Lantern

object
D
signal lantern
lighting hidden middle state

照亮流程中间状态的小型状态灯。

有记忆点，适合诊断和可见性，但比工程实践文章需要的气质更诗意。

Rollback Shuttle

object
E
loop-arrow shuttle
returning to checkpoint node

把失败工作流带回安全点的小型回退舱。

很适合回退路径主题，但长期使用会过度绑定“失败恢复”。

Trace Cartographer

abstract creature
F
map-like trace node
drawing a workflow route

把一次工作流运行画成可读路线的轨迹制图员。

隐喻很强，但缩小到社交卡片时识别度不如 Circuit Guide。
