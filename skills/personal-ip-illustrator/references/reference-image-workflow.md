# Reference Image Workflow

Use this when the user uploads or links images for inspiration, a rough character idea, a preferred style, or an existing IP they own.

## Intake

Ask what the reference image means if it is ambiguous:

- **Keep**: traits the user wants to preserve.
- **Avoid**: traits the user does not want copied.
- **Role**: whether the image is a style reference, character reference, color reference, mood reference, or composition reference.

If the user does not answer, infer conservatively and label assumptions.

## Extraction

Extract reusable traits, not a literal copy:

- silhouette and proportions
- face/front design
- palette
- materials or texture
- signature objects
- expression range
- style rules
- composition habits
- traits to avoid for originality

## Candidate Generation From References

When creating candidates from uploaded references:

- Generate several variations that preserve the chosen traits but change enough details to create an original IP.
- Keep each candidate visually comparable: similar framing, similar scale, neutral background.
- Include at least one content-use pose, not only a character turnaround.
- If a reference looks like an existing copyrighted character or another creator's IP, abstract the traits and avoid copying protected identifiers.

## User Confirmation

Do not ask the user to approve a long description. Show:

1. Candidate images or image prompts.
2. A short caption for each candidate.
3. A simple choice request: choose one, combine two, or request changes.

Useful feedback prompts:

- Which one is closest?
- What should stay?
- What should disappear?
- Should it feel more professional, playful, sharp, warm, strange, minimal, or technical?

## Finalization

After visual confirmation:

- Create a user-facing IP Card.
- Create the internal IP Bible.
- Test the IP with 2-3 different scene prompts before exporting a personal skill.
